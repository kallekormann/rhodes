# 21 вЂ” Internationalization (i18n)

**Status:** draft

## Context

Rhodes targets international teams. UI and AI outputs must support multiple languages without fragmenting the codebase.

## Decision

- **Primary language:** English (`en`)
- **Supported locales:** `en`, `es`, `de`, `fr`, `it`
- User-generated content is **not** auto-translated in V1 вЂ” only system UI and AI system messages

## Specification

### Frontend

| Aspect | Choice |
|--------|--------|
| Framework | `next-intl` (recommended for Next.js App Router) |
| Message files | `rhodes-app/messages/{locale}.json` |
| Default locale | `en` |
| Fallback | `de` в†’ `en`; all others в†’ `en` |
| Detection | User profile setting > browser `Accept-Language` > `en` |
| Persistence | `users.locale` column + localStorage cache |

### What gets translated (V1)

- Header labels, buttons, toasts, errors
- Sidebar section titles
- Template names and descriptions
- Email templates (Knowledge Bridge, invites)
- Onboarding hints

### What does NOT get translated (V1)

- User document content
- Library source text
- Imported PDF content

### Formatting

```typescript
// Dates
new Intl.DateTimeFormat(locale, { dateStyle: 'medium' }).format(date)

// Numbers (relevance %)
new Intl.NumberFormat(locale, { style: 'percent', maximumFractionDigits: 0 }).format(score)
```

RTL (Arabic, Hebrew): not in V1.

### LLM / AI language behavior

| Feature | Language rule |
|---------|---------------|
| Ingestion summary | Detected document language |
| "Why relevant?" one-liner | Document language, else user locale |
| Chat responses | Document language, else user locale |
| System prompts | Locale-specific template files |

### Language detection

1. **On ingest:** `franc` or LLM classify first 500 chars of extracted text
2. **On write:** re-detect when `content_plain` changes significantly (>20% diff)
3. Store on document: `documents.detected_language` (ISO 639-1)

### Embeddings and multilingual search

- **V1 model:** `nomic-embed-text` (768D) вЂ” reasonable multilingual performance
- **Evaluate for V1.1:** `mxbai-embed-large` (1024D) or `bge-m3` if DE/ES/FR/IT retrieval quality insufficient
- Semantic search is cross-lingual to a degree; no separate index per language in V1

### Email i18n

Knowledge Bridge emails use `users.locale` to select template. Unsubscribe link is locale-agnostic.
